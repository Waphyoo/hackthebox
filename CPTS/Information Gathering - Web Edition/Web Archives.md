# Web Archives (การจัดเก็บถาวรเว็บ)

## บทนำ

ในโลกดิจิทัลที่เปลี่ยนแปลงอย่างรวดเร็ว เว็บไซต์เกิดขึ้นและหายไป ทิ้งเพียงร่องรอยชั่วคราวของการมีอยู่ไว้เบื้องหลัง อย่างไรก็ตาม ด้วย **Wayback Machine** ของ Internet Archive เรามีโอกาสพิเศษที่จะย้อนกลับไปในอดีตและสำรวจร่องรอยดิจิทัลของเว็บไซต์ตามที่เคยเป็นมา

## Wayback Machine คืออะไร?

**Wayback Machine** คือคลังดิจิทัลของ World Wide Web และข้อมูลอื่นๆ บนอินเทอร์เน็ต ก่อตั้งโดย **Internet Archive** ซึ่งเป็นองค์กรไม่แสวงหาผลกำไร และเริ่มจัดเก็บถาวรเว็บไซต์ตั้งแต่ปี **1996**

### คุณสมบัติหลัก:

- **ย้อนเวลากลับไป**: ให้ผู้ใช้สามารถดูภาพรวม (snapshots) ของเว็บไซต์ตามที่ปรากฏในจุดต่างๆ ของประวัติศาสตร์
- **Captures/Archives**: ภาพรวมเหล่านี้ให้มุมมองของเวอร์ชันเก่าของเว็บไซต์ รวมถึงการออกแบบ เนื้อหา และการทำงาน

## วิธีการทำงานของ Wayback Machine

Wayback Machine ทำงานโดยใช้ **web crawlers** เพื่อจับภาพรวมของเว็บไซต์โดยอัตโนมัติในช่วงเวลาที่สม่ำเสมอ

### กระบวนการทำงาน 3 ขั้นตอน:

```
[Crawling] → [Archiving] → [Accessing]
```

#### 1. Crawling (การคลาน)

| องค์ประกอบ | คำอธิบาย |
|-----------|----------|
| **Web Crawlers/Bots** | ใช้บอทอัตโนมัติเพื่อเรียกดูอินเทอร์เน็ตอย่างเป็นระบบ |
| **การทำงาน** | บอทติดตามลิงก์จากหน้าเว็บหนึ่งไปยังอีกหน้าหนึ่ง เหมือนกับการคลิกไฮเปอร์ลิงก์เพื่อสำรวจเว็บไซต์ |
| **การดาวน์โหลด** | แทนที่จะเพียงอ่านเนื้อหา บอทเหล่านี้ดาวน์โหลดสำเนาของหน้าเว็บที่พบ |

#### 2. Archiving (การจัดเก็บถาวร)

| องค์ประกอบ | คำอธิบาย |
|-----------|----------|
| **การเก็บข้อมูล** | หน้าเว็บที่ดาวน์โหลดพร้อมทรัพยากรที่เกี่ยวข้อง (รูปภาพ, stylesheets, scripts) ถูกเก็บในคลังข้อมูลขนาดใหญ่ของ Wayback Machine |
| **Timestamp** | แต่ละหน้าเว็บที่จับภาพจะเชื่อมโยงกับวันที่และเวลาเฉพาะ สร้างภาพรวมทางประวัติศาสตร์ของเว็บไซต์ในช่วงเวลานั้น |
| **ความถี่** | กระบวนการจัดเก็บนี้เกิดขึ้นในช่วงเวลาที่สม่ำเสมอ บางครั้งรายวัน รายสัปดาห์ หรือรายเดือน ขึ้นอยู่กับความนิยมและความถี่ของการอัปเดตของเว็บไซต์ |

#### 3. Accessing (การเข้าถึง)

| องค์ประกอบ | คำอธิบาย |
|-----------|----------|
| **การเข้าถึงผ่าน Interface** | ผู้ใช้สามารถเข้าถึงภาพรวมที่เก็บถาวรผ่านอินเทอร์เฟซของ Wayback Machine |
| **การค้นหา** | ป้อน URL ของเว็บไซต์และเลือกวันที่เพื่อดูว่าเว็บไซต์ดูเป็นอย่างไรในจุดเวลานั้น |
| **เครื่องมือ** | มีเครื่องมือค้นหาคำเฉพาะภายในเนื้อหาที่เก็บถาวร หรือดาวน์โหลดเว็บไซต์ที่เก็บถาวรทั้งหมดเพื่อวิเคราะห์แบบออฟไลน์ |

### ความถี่ในการจัดเก็บ

| ปัจจัยที่มีอิทธิพล | ผลกระทบ |
|------------------|---------|
| **ความนิยมของเว็บไซต์** | เว็บไซต์ยอดนิยมอาจถูกจัดเก็บหลายครั้งต่อวัน |
| **อัตราการเปลี่ยนแปลง** | เว็บไซต์ที่อัปเดตบ่อยได้รับการจัดเก็บบ่อยขึ้น |
| **ทรัพยากรที่มี** | ขึ้นอยู่กับทรัพยากรที่ Internet Archive มี |
| **การจัดเก็บทั่วไป** | บางเว็บไซต์อาจมีเพียงไม่กี่ภาพรวมกระจายอยู่ตลอดหลายปี |

### ข้อจำกัด:

- **ไม่ครอบคลุมทั้งหมด**: Wayback Machine ไม่จับภาพทุกหน้าเว็บออนไลน์
- **การจัดลำดับความสำคัญ**: จัดลำดับความสำคัญของเว็บไซต์ที่ถือว่ามีคุณค่าทางวัฒนธรรม ประวัติศาสตร์ หรือการวิจัย
- **การยกเว้น**: เจ้าของเว็บไซต์สามารถขอให้เนื้อหาของตนถูกยกเว้นจาก Wayback Machine แม้ว่าจะไม่รับประกันเสมอไป

## ทำไม Wayback Machine จึงสำคัญต่อการสอดแนมเว็บ

Wayback Machine เป็นขุมทรัพย์สำหรับการสอดแนมเว็บ นำเสนอข้อมูลที่สามารถเป็นเครื่องมือสำคัญในสถานการณ์ต่างๆ ความสำคัญอยู่ที่ความสามารถในการเปิดเผยอดีตของเว็บไซต์ ให้ข้อมูลเชิงลึกที่มีค่าซึ่งอาจไม่ปรากฏชัดเจนในสถานะปัจจุบัน

### 1. เปิดเผยทรัพย์สินและช่องโหว่ที่ซ่อนอยู่ (Uncovering Hidden Assets and Vulnerabilities)

| การใช้งาน | รายละเอียด |
|-----------|----------|
| **ค้นพบทรัพยากรเก่า** | หน้าเว็บเก่า ไดเรกทอรี ไฟล์ หรือซับโดเมนที่อาจไม่สามารถเข้าถึงได้บนเว็บไซต์ปัจจุบัน |
| **เปิดเผยข้อมูลสำคัญ** | อาจพบข้อมูลที่สำคัญหรือช่องโหว่ด้านความปลอดภัยที่ถูกซ่อนไว้ |
| **ตัวอย่าง** | ค้นพบหน้า `/admin` เก่าที่ไม่มีในเว็บไซต์ปัจจุบัน |

### 2. ติดตามการเปลี่ยนแปลงและระบุรูปแบบ (Tracking Changes and Identifying Patterns)

| การใช้งาน | รายละเอียด |
|-----------|----------|
| **เปรียบเทียบภาพรวม** | เปรียบเทียบภาพรวมในอดีตเพื่อสังเกตว่าเว็บไซต์พัฒนาอย่างไร |
| **การเปลี่ยนแปลง** | เปิดเผยการเปลี่ยนแปลงในโครงสร้าง เนื้อหา เทคโนโลยี และช่องโหว่ที่อาจเกิดขึ้น |
| **ตัวอย่าง** | เห็นว่าเว็บไซต์เคยใช้ CMS เก่าที่มีช่องโหว่ที่ทราบ |

### 3. รวบรวมข่าวกรอง (Gathering Intelligence)

| การใช้งาน | รายละเอียด |
|-----------|----------|
| **แหล่งข้อมูล OSINT** | เนื้อหาที่เก็บถาวรเป็นแหล่งข้อมูล OSINT ที่มีค่า |
| **ข้อมูลเชิงลึก** | ให้ข้อมูลเชิงลึกเกี่ยวกับกิจกรรมในอดีตของเป้าหมาย กลยุทธ์ทางการตลาด พนักงาน และตัวเลือกเทคโนโลยี |
| **ตัวอย่าง** | ค้นพบรายชื่อพนักงานเก่าหรือข้อมูลติดต่อที่อาจใช้ใน social engineering |

### 4. การสอดแนมอย่างแอบซ่อน (Stealthy Reconnaissance)

| การใช้งาน | รายละเอียด |
|-----------|----------|
| **การเข้าถึงแบบ Passive** | การเข้าถึงภาพรวมที่เก็บถาวรเป็นกิจกรรมแบบ passive ที่ไม่โต้ตอบโดยตรงกับโครงสร้างพื้นฐานของเป้าหมาย |
| **ความปลอดภัย** | เป็นวิธีที่ตรวจจับได้น้อยกว่าในการรวบรวมข้อมูล |
| **ข้อดี** | ไม่ทิ้งร่องรอยบนเซิร์ฟเวอร์ของเป้าหมาย |

## การใช้งาน Wayback Machine กับ HackTheBox

### ตัวอย่างการใช้งาน:

เราสามารถดูเวอร์ชันที่เก็บถาวรแรกของ HackTheBox ได้โดย:

1. **ป้อน URL** ของหน้าที่ต้องการค้นหาใน Wayback Machine
2. **เลือกวันที่จับภาพที่เก่าที่สุด** ที่มีอยู่

**ตัวอย่าง:** วันที่เก็บถาวรแรกสุดของ HackTheBox คือ **2017-06-10 @ 04:23:01**

### ขั้นตอนการใช้งาน:

| ขั้นตอน | การดำเนินการ |
|---------|--------------|
| 1. **เข้า Wayback Machine** | ไปที่ https://web.archive.org |
| 2. **ป้อน URL** | ใส่ URL ของเว็บไซต์ที่ต้องการค้นหา |
| 3. **เลือกวันที่** | คลิกบนปฏิทินเพื่อเลือกวันที่ที่ต้องการดู |
| 4. **สำรวจ** | เรียกดูเว็บไซต์ในอดีตและสังเกตการเปลี่ยนแปลง |

## ประโยชน์ในการสอดแนม

### ตารางสรุปการใช้งาน:

| วัตถุประสงค์ | วิธีการ | ผลลัพธ์ที่คาดหวัง |
|-------------|--------|-----------------|
| **ค้นหาไฟล์เก่า** | ค้นหา snapshots ที่มีไฟล์หรือไดเรกทอรีที่ไม่มีในปัจจุบัน | เอกสารสำคัญ, ไฟล์คอนฟิก, โค้ดเก่า |
| **วิเคราะห์เทคโนโลยี** | ดูการเปลี่ยนแปลงของเทคโนโลยีที่ใช้ | ระบุเทคโนโลยีเก่าที่อาจมีช่องโหว่ |
| **รวบรวมข้อมูลพนักงาน** | ค้นหาหน้า "About Us" หรือ "Team" เก่า | รายชื่อพนักงาน, โครงสร้างองค์กร |
| **ติดตามโครงสร้าง** | เปรียบเทียบโครงสร้าง URL เมื่อเวลาผ่านไป | ค้นพบ endpoints ที่ถูกลืม, API เก่า |
