# Search Engine Discovery (การค้นพบผ่านเสิร์ชเอนจิ้น)

## บทนำ

Search engines (เสิร์ชเอนจิ้น) ทำหน้าที่เป็นผู้นำทางในภูมิทัศน์อันกว้างใหญ่ของอินเทอร์เน็ต ช่วยเราสำรวจผ่านพื้นที่ข้อมูลที่ดูเหมือนไม่มีที่สิ้นสุด อย่างไรก็ตาม นอกเหนือจากหน้าที่หลักในการตอบคำถามในชีวิตประจำวันแล้ว เสิร์ชเอนจิ้นยังเก็บขุมทรัพย์ของข้อมูลที่มีค่าอย่างยิ่งสำหรับการสอดแนมเว็บและการรวบรวมข้อมูล

แนวทางปฏิบัตินี้เรียกว่า **Search Engine Discovery** หรือ **OSINT (Open Source Intelligence) Gathering** เกี่ยวข้องกับการใช้เสิร์ชเอนจิ้นเป็นเครื่องมือที่ทรงพลังในการเปิดเผยข้อมูลเกี่ยวกับเว็บไซต์เป้าหมาย องค์กร และบุคคล

โดยการใช้ **search operators พิเศษ** เทคนิค และเครื่องมือ ผู้เชี่ยวชาญด้านความปลอดภัยและนักวิจัยสามารถเจาะลึกเข้าไปในเว็บที่ถูกจัดทำดัชนี เปิดเผยทุกอย่างตั้งแต่ข้อมูลพนักงานและเอกสารสำคัญ ไปจนถึงหน้าล็อกอินที่ซ่อนอยู่และข้อมูลประจำตัวที่ถูกเปิดเผย

## ทำไม Search Engine Discovery จึงสำคัญ

### เหตุผลหลัก:

| เหตุผล | คำอธิบาย |
|--------|----------|
| **Open Source** | ข้อมูลที่รวบรวมเป็นข้อมูลสาธารณะ ทำให้เป็นวิธีที่ถูกกฎหมายและมีจริยธรรมในการรับข้อมูลเชิงลึกเกี่ยวกับเป้าหมาย |
| **ความกว้างของข้อมูล** | เสิร์ชเอนจิ้นจัดทำดัชนีส่วนใหญ่ของเว็บ นำเสนอแหล่งข้อมูลที่เป็นไปได้ในวงกว้าง |
| **ง่ายต่อการใช้งาน** | เสิร์ชเอนจิ้นใช้งานง่าย ไม่ต้องใช้ทักษะทางเทคนิคเฉพาะทาง |
| **ประหยัดต้นทุน** | เป็นทรัพยากรฟรีและพร้อมใช้งานสำหรับการรวบรวมข้อมูล |

### การประยุกต์ใช้งาน:

| การใช้งาน | คำอธิบาย |
|-----------|----------|
| **Security Assessment** | ระบุช่องโหว่ ข้อมูลที่ถูกเปิดเผย และเวกเตอร์การโจมตีที่เป็นไปได้ |
| **Competitive Intelligence** | รวบรวมข้อมูลเกี่ยวกับผลิตภัณฑ์ บริการ และกลยุทธ์ของคู่แข่ง |
| **Investigative Journalism** | เปิดเผยการเชื่อมโยงที่ซ่อนอยู่ ธุรกรรมทางการเงิน และแนวทางปฏิบัติที่ไม่มีจริยธรรม |
| **Threat Intelligence** | ระบุภัยคุกคามที่เกิดขึ้นใหม่ ติดตามผู้กระทำที่เป็นอันตราย และคาดการณ์การโจมตีที่อาจเกิดขึ้น |

### ข้อจำกัด:

เสิร์ชเอนจิ้นไม่ได้จัดทำดัชนีข้อมูลทั้งหมด และข้อมูลบางอย่างอาจถูกซ่อนหรือปกป้องโดยเจตนา

## Search Operators (ตัวดำเนินการค้นหา)

Search operators เป็นเหมือนรหัสลับของเสิร์ชเอนจิ้น คำสั่งพิเศษและตัวปรับแต่งเหล่านี้ปลดล็อกระดับความแม่นยำและการควบคุมใหม่ ช่วยให้คุณระบุประเภทข้อมูลเฉพาะท่ามกลางความกว้างใหญ่ของเว็บที่ถูกจัดทำดัชนี

### ตาราง Search Operators พื้นฐานและขั้นสูง:

| Operator | คำอธิบาย | ตัวอย่าง | คำอธิบายตัวอย่าง |
|----------|----------|----------|-----------------|
| **site:** | จำกัดผลลัพธ์ให้เฉพาะเว็บไซต์หรือโดเมนที่ระบุ | `site:example.com` | ค้นหาหน้าที่เข้าถึงได้สาธารณะทั้งหมดบน example.com |
| **inurl:** | ค้นหาหน้าที่มีคำเฉพาะใน URL | `inurl:login` | ค้นหาหน้าล็อกอินบนเว็บไซต์ใดก็ได้ |
| **filetype:** | ค้นหาไฟล์ประเภทเฉพาะ | `filetype:pdf` | ค้นหาเอกสาร PDF ที่ดาวน์โหลดได้ |
| **intitle:** | ค้นหาหน้าที่มีคำเฉพาะในชื่อ | `intitle:"confidential report"` | ค้นหาเอกสารที่มีชื่อว่า "confidential report" หรือรูปแบบคล้ายกัน |
| **intext:** / **inbody:** | ค้นหาคำภายในเนื้อหาของหน้า | `intext:"password reset"` | ระบุหน้าเว็บที่มีคำว่า "password reset" |
| **cache:** | แสดงเวอร์ชันแคชของหน้าเว็บ (ถ้ามี) | `cache:example.com` | ดูเวอร์ชันแคชของ example.com เพื่อดูเนื้อหาก่อนหน้า |
| **link:** | ค้นหาหน้าที่ลิงก์ไปยังหน้าเว็บเฉพาะ | `link:example.com` | ระบุเว็บไซต์ที่ลิงก์ไปยัง example.com |
| **related:** | ค้นหาเว็บไซต์ที่เกี่ยวข้องกับหน้าเว็บเฉพาะ | `related:example.com` | ค้นพบเว็บไซต์ที่คล้ายกับ example.com |
| **info:** | ให้สรุปข้อมูลเกี่ยวกับหน้าเว็บ | `info:example.com` | รับรายละเอียดพื้นฐานเกี่ยวกับ example.com |
| **define:** | ให้คำนิยามของคำหรือวลี | `define:phishing` | รับคำนิยามของ "phishing" จากแหล่งต่างๆ |
| **numrange:** | ค้นหาตัวเลขภายในช่วงเฉพาะ | `site:example.com numrange:1000-2000` | ค้นหาหน้าบน example.com ที่มีตัวเลขระหว่าง 1000-2000 |
| **allintext:** | ค้นหาหน้าที่มีคำที่ระบุทั้งหมดในเนื้อหา | `allintext:admin password reset` | ค้นหาหน้าที่มีทั้ง "admin" และ "password reset" |
| **allinurl:** | ค้นหาหน้าที่มีคำที่ระบุทั้งหมดใน URL | `allinurl:admin panel` | ค้นหาหน้าที่มี "admin" และ "panel" ใน URL |
| **allintitle:** | ค้นหาหน้าที่มีคำที่ระบุทั้งหมดในชื่อ | `allintitle:confidential report 2023` | ค้นหาหน้าที่มี "confidential," "report," และ "2023" ในชื่อ |
| **AND** | จำกัดผลลัพธ์โดยต้องมีคำทั้งหมด | `site:example.com AND (inurl:admin OR inurl:login)` | ค้นหาหน้า admin หรือ login บน example.com |
| **OR** | ขยายผลลัพธ์โดยรวมหน้าที่มีคำใดคำหนึ่ง | `"linux" OR "ubuntu" OR "debian"` | ค้นหาหน้าที่กล่าวถึง Linux, Ubuntu หรือ Debian |
| **NOT** | ยกเว้นผลลัพธ์ที่มีคำที่ระบุ | `site:bank.com NOT inurl:login` | ค้นหาหน้าบน bank.com ยกเว้นหน้าล็อกอิน |
| **\*** (wildcard) | แทนอักขระหรือคำใดก็ได้ | `site:socialnetwork.com filetype:pdf user* manual` | ค้นหาคู่มือผู้ใช้ (user guide, user handbook) ในรูปแบบ PDF |
| **..** (range search) | ค้นหาผลลัพธ์ในช่วงตัวเลขที่ระบุ | `site:ecommerce.com "price" 100..500` | ค้นหาสินค้าที่มีราคาระหว่าง 100-500 |
| **" "** (quotation marks) | ค้นหาวลีที่ตรงทุกประการ | `"information security policy"` | ค้นหาเอกสารที่มีวลี "information security policy" แบบตรงทุกตัว |
| **-** (minus sign) | ยกเว้นคำจากผลการค้นหา | `site:news.com -inurl:sports` | ค้นหาบทความข่าวบน news.com ยกเว้นเนื้อหาเกี่ยวกับกีฬา |

## Google Dorking

**Google Dorking** หรือที่เรียกว่า **Google Hacking** คือเทคนิคที่ใช้ประโยชน์จากพลังของ search operators เพื่อเปิดเผยข้อมูลสำคัญ ช่องโหว่ด้านความปลอดภัย หรือเนื้อหาที่ซ่อนอยู่บนเว็บไซต์โดยใช้ Google Search

### ตัวอย่าง Google Dorks ทั่วไป:

#### 1. การค้นหาหน้าล็อกอิน (Finding Login Pages)

| Google Dork | จุดประสงค์ |
|-------------|-----------|
| `site:example.com inurl:login` | ค้นหาหน้าล็อกอินบน example.com |
| `site:example.com (inurl:login OR inurl:admin)` | ค้นหาหน้าล็อกอินหรือแอดมินบน example.com |

#### 2. การระบุไฟล์ที่ถูกเปิดเผย (Identifying Exposed Files)

| Google Dork | จุดประสงค์ |
|-------------|-----------|
| `site:example.com filetype:pdf` | ค้นหาไฟล์ PDF บน example.com |
| `site:example.com (filetype:xls OR filetype:docx)` | ค้นหาไฟล์ Excel หรือ Word บน example.com |

#### 3. การเปิดเผยไฟล์คอนฟิก (Uncovering Configuration Files)

| Google Dork | จุดประสงค์ |
|-------------|-----------|
| `site:example.com inurl:config.php` | ค้นหาไฟล์คอนฟิก PHP บน example.com |
| `site:example.com (ext:conf OR ext:cnf)` | ค้นหานามสกุลไฟล์ที่ใช้สำหรับไฟล์คอนฟิกทั่วไป |

#### 4. การค้นหาการสำรองฐานข้อมูล (Locating Database Backups)

| Google Dork | จุดประสงค์ |
|-------------|-----------|
| `site:example.com inurl:backup` | ค้นหาไฟล์สำรองบน example.com |
| `site:example.com filetype:sql` | ค้นหาไฟล์ SQL (อาจเป็นการสำรองฐานข้อมูล) |

### แหล่งข้อมูลเพิ่มเติม:

สำหรับตัวอย่างเพิ่มเติม สามารถดูได้ที่ **Google Hacking Database** ซึ่งรวบรวม Google Dorks มากมายสำหรับวัตถุประสงค์ต่างๆ
